Test cases

Sample test cases from manual and automated testing practice

About this project: Test cases written for the login and checkout flow of a demo trading platform, covering positive, negative, and edge-case scenarios.
Login — email and password authentication
ID Title Steps Expected result Status
TC-001 Successful login with valid credentials 1. Open login page
2. Enter valid email and password
3. Click "Log in"		 User is redirected to the dashboard Pass
TC-002 Login with incorrect password 1. Open login page
2. Enter valid email, wrong password
3. Click "Log in"		 Error message "Incorrect email or password" is shown, user stays on login page Pass
TC-003 Login with empty fields 1. Open login page
2. Leave email and password empty
3. Click "Log in"		 "Log in" button is disabled, or validation errors appear under both fields Fail
TC-004 Login with SQL injection attempt in email field 1. Open login page
2. Enter ' OR '1'='1 in email field
3. Click "Log in"		 Input is safely rejected, generic error message is shown, no database error is exposed Pass
Checkout — payment step
ID Title Steps Expected result Status
TC-101 Checkout with valid card details 1. Add item to cart
2. Proceed to checkout
3. Enter valid card details
4. Click "Pay now"		 Payment succeeds, confirmation page is displayed Pass
TC-102 Checkout with expired card 1. Add item to cart
2. Proceed to checkout
3. Enter card with past expiry date
4. Click "Pay now"		 Error "Card has expired" is shown, payment is not processed Blocked
TC-103 Cart total updates after removing an item 1. Add two items to cart
2. Remove one item
3. Check total price		 Total price recalculates correctly after removal Pass
